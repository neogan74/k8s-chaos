## Problems Found

### P0 — Critical / Blocking

- **`e2e-f-pod-network-loss` branch is not merged.**
  The branch is clean (no uncommitted changes) and all unit tests pass,
  but it has not been PR'd into `main`. It carries:
  - E2E tests for `pod-network-loss` (basic, safety, validation, advanced)
  - A significant error-handling refactor across the controller, webhook,
    and CLI commands (`stats`, `top`, `describe`, `list`)
  - New `internal/controller/errors.go` with `247` lines of structured
    error types and a `PermissionDenied` handling path
  - CI workflow improvements (branch filters, golangci-lint v7, pinned v2.1.6)
  This is the single biggest blocker for ongoing work because any new
  feature branch off `main` will re-introduce conflicts.

### P1 — Important

- **Test coverage far below the 80% target.**
  As measured today with `go test ./... -cover`:

  | Package | Coverage |
  |---|---|
  | `api/v1alpha1` | 44.2% |
  | `internal/controller` | 33.9% |
  | `pkg/cli/cmd` | 19.3% |
  | `internal/metrics` | 0.0% |
  | `cmd/k8s-chaos-cli` | 0.0% |

  The most critical gap is `internal/controller` — 3 712-line file with
  ~66% of logic untested. No CI coverage gate enforces the threshold, so
  regressions are silent.

- **`internal/controller/chaosexperiment_controller.go` is a god file.**
  At 3 712 lines it covers pod kill, delay, CPU/memory stress,
  network loss/corruption, disk fill, node drain/taint/cpu-stress, history
  management, scheduling, and retry logic all in one file. This makes
  targeted testing and code review difficult. Refactoring into per-action
  handler files (matching the pattern of `errors.go`) is needed.

- **Audit actor not captured in history (`history.go:291`).**
  `// TODO: Extract from request context` — the requesting user is never
  recorded in experiment history. Audit logging is listed as complete in
  TODO.md and ROADMAP.md but this detail is missing.

- **`pod-disk-fill` controller not implemented.**
  ADR 0008 and docs are complete. CRD fields are designed. Controller
  logic, webhook validation, and tests are all still pending. This is the
  last unimplemented node/pod chaos action in the core set.

- **Stale local branch sprawl.**
  Local branches `doc`, `errors`, `history-ttl`, `node-taint-handle`,
  `вp-mgmnt` (Cyrillic typo) appear merged or superseded. These add noise
  and slow down `git branch` / checkout autocompletion. Remote branches
  `origin/Emit-Kubernetes-Events`, `origin/k8s-events`, `origin/one`,
  `origin/shed`, `origin/err_term` also look stale.

### P2 — Minor

- **`network-partition` remote branch exists but is not integrated.**
  `origin/network-partition` is present but no local tracking branch or
  open PR. If experimental, it should either be picked up or closed.

- **`pod-network-corruption` not merged into main.**
  Local and remote branches exist; feature appears complete from prior
  backlog but never landed in `main`.

- **Documentation drift.**
  `E2E_FIXES_SUMMARY.md` and `E2E_TESTS_FIX.md` in repo root contain
  ephemeral fix notes that duplicate ADR/CHANGELOG content and will
  become misleading as the code evolves. Candidates for deletion.

- **`Pause/resume` spec field is absent.**
  Listed as P1 in `backlog.md` and in TODO.md for months. No spike or ADR
  started. Controller needs a guard at reconcile entry to skip work when
  `spec.paused == true` and emit an appropriate status condition.

## Plan for Next Week (2026-06-23 → 2026-06-29)

### Must Do

1. **Merge `e2e-f-pod-network-loss` into `main`.**
   Open a PR, run CI, merge. This unblocks all downstream feature branches
   and closes the P0 "Add E2E coverage for `pod-network-loss`" item.

2. **Implement `pod-disk-fill` controller.**
   Follow ADR 0008 design: ephemeral container with `dd`/`fallocate`,
   `fillPercentage` + `targetPath` + `volumeName` spec fields, cleanup on
   experiment end, safety wiring (dry-run, exclusion labels, maxPercentage),
   unit tests. Update webhook validation and sample config.

3. **Add `paused: true` spec field and controller guard.**
   Add `Paused bool` to `ChaosExperimentSpec`. In `Reconcile`, if
   `spec.paused` is true, set status condition `Paused` and return without
   requeue. Resume resumes the normal reconcile cycle. Add webhook
   validation and unit tests.

4. **Prune stale branches.**
   Delete confirmed-merged local branches: `doc`, `history-ttl`,
   `node-taint-handle`, `errors`, `вp-mgmnt`. Close or pick up the
   `pod-network-corruption` branch from the remote.

### New Features to Implement

5. **CLI `validate` command.**
   Validate a local experiment YAML file: schema check, cross-field rules
   (duration required for delay/stress, lossPercentage range, etc.),
   optional live cluster checks (namespace exists, selector matches pods).
   Feeds into the CI/CD use-case of "validate before apply".

6. **CLI `create` interactive wizard.**
   Guided `k8s-chaos create` that prompts for action, namespace, label
   selector, count, and duration; shows a YAML preview; and applies on
   confirmation. Follows the existing `cobra` command structure.

7. **Resolve audit actor TODO in `history.go:291`.**
   Thread the `UserAgent` / user identity from the admission request
   context into history records when experiments are created or updated
   via the webhook. Store in `HistoryEntry.Actor` (new field).

### Stretch

8. **Add CI coverage threshold gate.**
   Fail the `test` workflow if `internal/controller` coverage drops below
   40% (stepping stone toward 80%). Use `go test -coverprofile` + `go tool
   cover -func` with an `awk` threshold check.

9. **Split controller into per-action handler files.**
   Extract `handlePodNetworkLoss`, `handlePodDiskFill`, `handleNodeTaint`,
   `handleNodeCPUStress` into `internal/controller/handler_*.go` files.
   Keeps the main controller focused on reconcile orchestration only.
